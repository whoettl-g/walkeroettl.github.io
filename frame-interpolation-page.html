Anime Frame Interpolation with U-Net

IST 691: Deep Learning in Practice | Fall 2025

Python TensorFlow U-Net Computer Vision

Overview

Traditional animation is labor-intensive—animators draw keyframes and then painstakingly create the "in-between" frames that make motion fluid. This project explores whether a deep learning model can assist that process by generating intermediate frames automatically.

I built a U-Net convolutional neural network from scratch, trained specifically on Pokémon animation footage. The idea is to train on a single series' visual style so the model learns that specific aesthetic, rather than trying to generalize across all animation styles. Given two frames, the model predicts what the frame between them should look like.

Technical Approach

Frame Extraction Pipeline: Built a pipeline to extract training triplets from video using a 9-frame window (frames 1-9), saving the start frame, middle frame (5), and end frame (9). The model learns to predict the middle frame given the two endpoints. To maximize training data, the window slides with a stride of 5 frames—so triplets come from frames 1-9, then 6-14, then 11-19, and so on—creating overlapping samples for more robust training.

U-Net Architecture: Implemented a ~31 million parameter U-Net—an encoder-decoder architecture with skip connections that preserves spatial detail while learning semantic features. The model takes two frames as input (concatenated) and outputs the predicted intermediate frame.

Recursive Interpolation: The model generates frame 5 from keyframes 1 and 9, then recursively fills in the gaps—predicting frame 3 from frames 1 and 5, frame 7 from frames 5 and 9, and so on. This approach generates 7 intermediate frames from just 2 drawn keyframes, meaning animators would only need to draw roughly 1 frame for every 8 in the final output.

Scaling Considerations

As a class project, this was constrained by available compute and training data. A production-scale version would benefit significantly from more GPU hours for training, a larger and more diverse dataset from the target series, and potentially a more sophisticated architecture like a video diffusion model. The core approach—training on a specific animation style rather than generalizing—remains sound and could yield much higher quality results with additional resources.
